## 3. Dataset Plan

### 3.1 Dataset Requirements

Dataset must use Alpaca-style fields:

```json
{
  "instruction": "...",
  "input": "...",
  "output": "..."
}
```

Use 100–500 examples.

Recommended size:

```text
200–300 samples
```

Dataset options:

1. Use Vietnamese Alpaca-style dataset if available.
2. Or create a small custom Vietnamese instruction-following dataset for a focused domain.

Recommended custom domain:

```text
Vietnamese AI / coding tutor assistant
```

This is good for qualitative evaluation because prompts are easy to inspect.

### 3.2 Dataset Cleaning

Implement or verify these checks:

* Remove examples with empty `instruction` or `output`.
* Normalize missing `input` to empty string.
* Remove duplicate examples.
* Remove very short outputs.
* Keep only examples with reasonable length.
* Split dataset into train/eval using fixed seed `42`.
* Use 90% train and 10% eval.

### 3.3 Prompt Formatting

Use Alpaca-style prompt:

```text
### Instruction:
{instruction}

### Input:
{input}

### Response:
{output}
```

If `input` is empty, still keep a consistent template:

```text
### Instruction:
{instruction}

### Response:
{output}
```

Create a `text` column for SFT training.

### 3.4 Token Length Analysis

Before training:

* Tokenize all formatted examples.
* Compute min, mean, median, p90, p95, max token length.
* Set `max_seq_length` based on p95.
* Round to a safe value such as `512` or `1024`.

Save this analysis into `REPORT.md`.

Example expected statement:

```text
The p95 token length was around X tokens, so max_seq_length was set to Y to cover most samples while avoiding unnecessary VRAM usage.
```

## 4. Training Plan

### 4.1 Common Training Configuration

All rank experiments must use the same dataset, split, model, epochs, LR, batch size, and sequence length.

Use:

```python
num_train_epochs = 3
learning_rate = 2e-4
lr_scheduler_type = "cosine"
warmup_ratio = 0.10
optim = "adamw_8bit"
gradient_checkpointing = True
load_in_4bit = True
```

Batching:

```python
per_device_train_batch_size = 2
gradient_accumulation_steps = 4
effective_batch_size = 8
```

LoRA target modules baseline:

```python
target_modules = ["q_proj", "v_proj"]
```

LoRA dropout:

```python
lora_dropout = 0
```

Random seed:

```python
seed = 42
```

### 4.2 Rank Experiments

Train three adapters:

| Experiment | LoRA rank | LoRA alpha |
| ---------- | --------: | ---------: |
| low rank   |         8 |         16 |
| baseline   |        16 |         32 |
| high rank  |        64 |        128 |

For each run:

1. Load base model in 4-bit.
2. Attach LoRA adapter.
3. Count trainable parameters.
4. Reset CUDA peak memory stats.
5. Train.
6. Measure total training time.
7. Measure peak VRAM.
8. Evaluate on eval split.
9. Compute perplexity.
10. Save adapter.
11. Save metrics.

Pseudo-code:

```python
import time
import math
import torch

torch.cuda.reset_peak_memory_stats()
start_time = time.time()

trainer.train()

training_time_min = (time.time() - start_time) / 60
peak_vram_gb = torch.cuda.max_memory_allocated() / 1024**3

eval_metrics = trainer.evaluate()
eval_loss = eval_metrics["eval_loss"]
perplexity = math.exp(eval_loss)
```

Clear GPU between experiments:

```python
import gc
del model
del trainer
gc.collect()
torch.cuda.empty_cache()
```
